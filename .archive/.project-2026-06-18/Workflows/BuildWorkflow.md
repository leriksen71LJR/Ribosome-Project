# Build Workflow (Stateless Coordinator)

**Status:** Draft / Research  
**Purpose:** Primary coordinator workflow for significant work on Fortress.

See `.docs/Research/DocumentationAsWorkflow-Research.md` for the full model explanation.

This workflow is **stateless**. It defines the process the agent must follow. It does not carry state between builds.

---

## Process Steps (High Level)

1. **Initialization Gate**
2. **Task Understanding** (Task/Spec provided as input data)
3. **Implementation** (using defined Procedures)
4. **Mandatory Documentation Audit Gate**
5. **Violation Handling** (if needed)
6. **Final Output Verification**

Full details and enforcement rules are maintained in the research document for now. This file will be expanded in Phase 2.
