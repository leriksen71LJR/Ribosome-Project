# EvaluationCriteria.md

**Purpose**  
This document guides agents when performing a documentation audit. The goal is to understand how agents interpret the documentation, where it shapes their thinking, and what is missing.

**Restriction:**  
Agents may **only report** findings and reasoning. They are **not allowed** to edit any documentation files.

## Deep Audit Instructions

Think critically. Show your reasoning.

For each area, explain:
- How you interpreted the rules
- What mental model you built
- Where you had to make assumptions
- What felt unclear or contradictory
- How the documentation influenced your decisions
- What would have made your job significantly better

## Evaluation Areas

### 1. Mental Model Formation
- What overall mental model did you form about Fortress?
- How did the documentation help or hinder forming that model?
- Where did you have to fill in gaps yourself?

### 2. Clarity and Ambiguity
- Which parts were clear?
- Which parts were ambiguous or open to multiple interpretations?
- Where did you have to choose between competing understandings?

### 3. Completeness and Gaps
- What important guidance was missing?
- What edge cases were not addressed?
- What would have made the documentation dramatically more effective?

### 4. Consistency
- Did you find contradictions between documents?
- How did you resolve them?

### 5. Practical Effectiveness
- How well did the documentation support real implementation?
- What friction did you experience?
- What patterns of behavior did the documentation encourage?

## Required Output Format

Include a section titled **"Deep Documentation Audit"** with this structure:

**Deep Documentation Audit**

**Mental Model Formed**  
[Your understanding of the intended system]

**Strengths**  
[What worked well]

**Significant Issues**  
[Specific problems with references]

**Recommendations**  
[Constructive suggestions]

**Overall Assessment**  
[How effective was the documentation? Rate 1–10 and explain]
