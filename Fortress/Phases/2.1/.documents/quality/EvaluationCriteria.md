# Evaluation Criteria — Deep Documentation Audit

**Purpose:** Rubric for build report §5 (Deep Documentation Audit).

## Minimum evidence per Significant Issue

- Document path (under `006/`)
- Section or topic
- Problem description
- What was done instead
- Severity (High / Medium / Low)
- Resolution or impact

## Quality rubric

| Score | Meaning |
|-------|---------|
| 9–10 | Detailed, cited, actionable |
| 7–8 | Solid with minor vagueness |
| 5–6 | Some issues but missing citations |
| 1–4 | Generic — must be redone |

## DI-layout documents to evaluate

Evaluate each document you actually read from the README read order. Skip with justification if not consulted.

Include **test measurability:** cite `quality/TEST-EXPECTATIONS.md` checklist results in the audit or state why tests are partial.

Shallow audits fail the requirement.
