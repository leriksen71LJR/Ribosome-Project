# Naming-Terminology-Consistency.md

**Type:** Analytical Perspective (Strategy)

**Purpose:** Evaluate documentation for consistent use of terms, names, and concepts across files.

## Key Questions
- Are the same concepts referred to with the same names throughout the documentation?
- Do conflicting or drifting terms create ambiguity for agents or humans?

## Application Guidance
Use this perspective when reviewing documentation for clarity and to reduce the chance of agents making incorrect assumptions due to inconsistent terminology.

## Notes
Inconsistent naming was observed as a contributing factor to some interpretation issues during the Phase 1.2A shootout. This perspective helps surface such problems.
