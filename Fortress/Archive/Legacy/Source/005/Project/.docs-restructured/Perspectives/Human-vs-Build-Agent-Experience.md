# Human-vs-Build-Agent-Experience.md

**Type:** Analytical Perspective (Strategy)

**Purpose:** Evaluate documentation and proposals from both human developer and build agent perspectives.

## Key Questions
- How does this documentation or proposal affect humans reading and using it?
- How does it affect build agents trying to follow the specifications?
- Are there conflicts between the needs of these two audiences?

## Application Guidance
Use this perspective when designing or reviewing documentation structure, especially when balancing readability for humans with precision for agents.

## Notes
This perspective often highlights tensions between narrative explanation and executable pseudocode. It should be used alongside other perspectives such as Risk and Effort vs Impact.
