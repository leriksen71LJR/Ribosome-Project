# Process-vs-Requirements-Inversion.md

**Type:** Analytical Perspective (Strategy)

**Purpose:** Identify and evaluate situations where process documentation is stronger or more complete than requirements documentation.

## Key Observation from Phase 1.2A
Process documents (`AGENTS.md`, `BuildDisclosure.md`) consistently scored higher in executability than requirements documents (`CODING_DESIGN.md` gaps, incomplete contracts, deferred phrases).

## Application Guidance
Use this perspective to detect when effort is being spent on process rules to compensate for weak or incomplete requirements. The recommended priority is usually to strengthen requirements and contracts first.

## Notes
This perspective was strongly supported by agent disclosures in the Phase 1.2A shootout. It helps prioritize documentation improvement efforts.
