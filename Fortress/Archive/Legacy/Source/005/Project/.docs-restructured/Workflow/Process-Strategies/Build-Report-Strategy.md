# Build-Report-Strategy.md

**Type:** Process Strategy  
**Purpose:** Defines the required structure and content for build reports.

This is one possible strategy for how build reports should be created. Different strategies could be developed for different contexts or levels of formality.

## Required Sections

1. Summary of Work Completed
2. Files Created / Modified
3. Key Implementation Decisions (grouped by topic)
4. Deviations from Documentation
5. Deep Documentation Audit
6. Open Gaps and Assumptions
7. Recommended Next Steps

## Naming Convention

`BUILD-REPORT-YYYY-MM-DD-XXX.md` where `XXX` is a 3-digit daily sequence.

## Location

All build reports must be written to `.docs/Builds/`.

## Notes

This strategy can be replaced or extended by other strategies in `Workflow/Process-Strategies/` without modifying core rules.
