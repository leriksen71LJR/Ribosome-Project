# Deviation-Reporting-Strategy.md

**Type:** Process Strategy

**Purpose:** Define how and when deviations from documentation must be reported.

## Requirements
- Any inability to follow a documented rule must be explicitly reported in the build report under **Deviations from Documentation**.
- For each deviation, include:
  - The specific rule not followed
  - What was done instead
  - Why it was necessary
  - Impact on correctness, security, or maintainability

## Notes
This strategy supports honest self-reporting and is considered mandatory for all builds. Alternative deviation reporting approaches can be developed as separate strategies if needed.
