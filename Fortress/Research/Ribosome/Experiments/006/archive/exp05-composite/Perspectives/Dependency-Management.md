# Dependency-Management.md

**Type:** Analytical Perspective (Strategy)

**Purpose:** Evaluate documentation and proposals based on how well they manage dependencies between components and documents.

## Key Questions
- Does this documentation clearly define what depends on what?
- Are there hidden or unclear dependencies that could cause problems during implementation or evolution?

## Application Guidance
Use this perspective when reviewing or designing documentation that involves multiple components or cross-references. Clear dependency management reduces the risk of circular dependencies and implementation order violations.

## Notes
This perspective is closely related to the Implementation Order strategy and the Sequencing & Dependencies perspective.
