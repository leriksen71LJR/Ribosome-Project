# Edge-Cases-Failure-Modes.md

**Type:** Analytical Perspective (Strategy)

**Purpose:** Identify where documentation or implementation could break under unusual but realistic conditions.

## Key Questions
- Where could this break under high-ambiguity or high-stakes scenarios (e.g., security, data migration, unusual user behavior)?
- What happens in edge cases that are not explicitly covered?

## Application Guidance
Use this perspective when reviewing specifications, especially around security, storage, and exception handling. It helps surface hidden risks before they become problems in production.

## Notes
This perspective complements the Risk Perspective and is particularly useful during Deep Documentation Audits.
