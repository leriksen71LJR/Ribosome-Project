# EvaluationCriteria.md (Quality)

**Purpose:** Guide agents performing the mandatory Deep Documentation Audit in build reports.

This file focuses on the audit process and quality expectations.

## When This Applies
- Every build must include a Deep Documentation Audit section.
- Shallow audits fail the requirement.

## Minimum Evidence Requirements
Each Significant Issue must include:
- Document referenced
- Specific section or topic
- Problem description
- What was done instead
- Severity
- Resolution or impact

## Audit Quality Rubric
- 9–10: Detailed, cited, actionable
- 7–8: Solid with minor vagueness
- 5–6: Some issues but missing citations
- 1–4: Generic — must be redone

## Notes
This strategy can evolve as audit practices improve. New evaluation approaches can be added as separate strategies.
