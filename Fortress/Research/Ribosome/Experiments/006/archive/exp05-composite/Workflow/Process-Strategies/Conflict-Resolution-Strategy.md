# Conflict-Resolution-Strategy.md

**Type:** Process Strategy

**Purpose:** Define how to handle contradictions between documents (Rule 10).

## Process
1. Clearly identify both documents involved.
2. State the specific conflicting statements.
3. Declare which document is treated as authoritative and why.
4. Record the decision in the build report.

## Notes
This strategy supports transparent decision-making when documentation conflicts. It is considered mandatory. Alternative conflict resolution approaches can be developed as separate strategies if needed.
