# Pre-Build-Disclosure-Strategy.md

**Type:** Process Strategy (Deprecated for Standard Builds)

**Purpose:** Document that pre-build disclosure is deprecated for standard builds.

## Status
Pre-build disclosure runs are no longer recommended for routine builds. They can still be useful for greenfield documentation audits but are considered lower value once a package has been improved.

## Notes
Post-build disclosure (Rule 12) is the primary and recommended mechanism for surfacing documentation issues.
