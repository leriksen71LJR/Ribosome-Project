# Rule-10-11-Strategy.md

**Type:** Process Strategy

**Purpose:** Define how to handle document conflicts (Rule 10) and gap/assumption reporting (Rule 11).

## Rule 10 — Conflict Resolution
When documents conflict:
1. Clearly identify both documents.
2. State the specific conflicting statements.
3. Declare which document is treated as authoritative and why.
4. Document the decision in the build report.

## Rule 11 — Gap Reporting
When documentation is unclear, incomplete, or ambiguous:
- Flag the gap explicitly rather than making silent assumptions.
- For high-risk assumptions, provide justification and severity.
- Record assumptions in the build report under **Open Gaps and Assumptions**.

## Notes
This strategy supports honest and transparent reporting. It is considered mandatory.
