# Rule-3-Strategy.md

**Type:** Process Strategy

**Purpose:** Enforce early returns and guard clauses to minimize nesting (Rule 3 from coding standards).

## Requirements
- Prefer early returns and guard clauses over deep nesting.
- Keep classes and methods small and focused.
- Each class should have one clear reason to change.

## Notes
This strategy supports readability and maintainability. It aligns with defensive programming principles.
