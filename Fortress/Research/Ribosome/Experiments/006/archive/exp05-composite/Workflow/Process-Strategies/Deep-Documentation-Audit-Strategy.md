# Deep-Documentation-Audit-Strategy.md

**Type:** Process Strategy

**Purpose:** Define the requirements and quality expectations for the mandatory Deep Documentation Audit in every build report.

## When Required
- Every build must include a **Deep Documentation Audit** section.
- Shallow or generic audits are considered incomplete.

## Minimum Evidence Requirements
Each Significant Issue entry must include:
- Document referenced
- Specific section/topic
- Problem description
- What was done instead
- Severity
- Resolution or impact

## Quality Rubric
- 9–10: Detailed, well-cited, actionable recommendations
- 7–8: Solid with minor vagueness
- 5–6: Issues named but missing citations or severities
- 1–4: Generic — must be redone

## Notes
This strategy supports rigorous self-assessment of documentation quality. Alternative audit approaches can be developed as separate strategies.
