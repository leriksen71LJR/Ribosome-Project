# Sequencing-Dependencies.md

**Type:** Analytical Perspective (Strategy)

**Purpose:** Evaluate proposals and changes based on what must exist before they can work well and what should come after.

## Key Questions
- What dependencies or prerequisites does this change have?
- What should logically follow this change?
- Does this create a realistic and logical sequence of work?

## Application Guidance
Use this perspective when planning documentation improvements, new features, or structural changes. It helps create realistic roadmaps and avoid working out of dependency order.

## Notes
This perspective is especially useful when combined with the Implementation Order strategy.
