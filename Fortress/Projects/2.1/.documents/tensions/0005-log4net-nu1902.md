# ADR-0005: Log4Net NU1902 Policy

**Status:** Resolved (Phase 1.2A)  
**Seam:** S2 — dependency policy

## Context

Architecture mandates Log4Net. NuGet may report NU1902 (known vulnerability) on pinned versions.

## Decision

Use the Log4Net version that restores successfully for the assigned TFM. If NU1902 appears:

1. Document the exact package version in the build report.
2. State whether vulnerability is accepted for MVP or mitigated by usage scope (local console, no network appenders).
3. Do not silently replace Log4Net with another framework without deviation report.
