# Procedure: Documentation Audit

**Status:** Draft / Research

This procedure is called from `Workflows/BuildWorkflow.md`.

## Purpose
Force the agent to reflect deeply on how well the documentation guided the build.

## Steps
1. Read `.docs/Evaluation/EvaluationCriteria.md`
2. Perform a **Deep Documentation Audit** (not a shallow checklist)
3. Include a clearly labeled "Deep Documentation Audit" section in the build report
4. Be honest about ambiguities, gaps, and assumptions you had to make

## Enforcement
This is a **mandatory gate**. A build report without a proper Deep Documentation Audit section will be treated as incomplete.
